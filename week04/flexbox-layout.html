Web Development Standards Checklist

Overview

These standards focus feedback, encourage discipline, and mirror realistic team baselines. They draw in part from David Dias’s Front-End Checklist and MDN references.

Standards Checklist

  • All folders/files use lowercase, no spaces, meaningful names.
  • Document type declaration: <!DOCTYPE html>
  • <html lang="en-US">
  • Head: charset, viewport, title, description, author, external CSS.
  • HTML is valid (W3C validator). Semantic elements are used. No broken links.
  • CSS is valid (W3C validator). Prefer classes; IDs, if used, are unique.
  • Images optimized; aspect ratios preserved; meaningful alt text.
  • Exactly one <h1> that aligns with <title>.
  • Headings follow logical order; no AA color contrast violations.
  • Accessibility warnings reviewed; page weight ≤ 500 kB (excl. maps).
